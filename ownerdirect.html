Owner Direct

A property listing platform that connects landlords to tenants directly

Read Case Study

Overview

My goal was to design the platform in a clean and modern visual style making it easy for people and find properties, having obvious call to actions for property owners and people searching for property, and also communicating the value of the platform is important to the overall user experience.

ROLE IN PROJECT

User Experience Designer

DURATION

3 Weeks

The Problem

Most property listing websites available in Nigeria just load many lisings on their platform and do not properly segment available listings, this leads to information overload and makes it difficult for users to find properties. The goal was to;

1. Design a search experience that guides the user through searching through a location, property type, and selecting a budget.
2. Properly segment and display listings to minimise clutter and make it easy for users to find properties.

The design process

Since this was a freelance project, I had to come up with a timeline for the design. I estimated the time it would take to research and since a web version of the application already existed, I outlined possible screens of features and estimated a timeline to completed the design of these screens for mobile. Research

I spoke with stakeholders to better understand the goal of the Klassconnect learning tool. This helped me to better understand the features build into the web app and why those features existed. Next, I interviewed some students who use similar learning tools to know if they found such tools useful and what features they interact frequently with. I also explored other similar applications to understand how they work and to ascertain how best to visually represent existing features in to the mobile app. This was important to have a grasp of the current experience and this provided direction for the userflow, guided the design.

At the end of this stage in the design process, I had a clear understanding of features that exists on the web version and based off user interviews I prioritized what features are most important (the ones the users interact with frequently).

Conclusion

Simplicity was key to the design of the app because it had quite a number of features, so based on my research I ensured that the most used features were placed in a bottom navigation menu and so users can easily navigate through the most used features.

Other features were placed in an hamburger menu and each had a descriptive icon so it's easy for a user to find a feature by merely looking at the icons.
